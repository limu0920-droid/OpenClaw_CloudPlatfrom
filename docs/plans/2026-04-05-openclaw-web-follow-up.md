# OpenClaw Web Follow-up Plan

## 当前已完成

- Vue Portal/Admin 双布局
- Login 入口页
- Portal 实例列表、实例详情、任务、日志
- Admin 总览、租户、实例、任务、告警、审计
- Go Mock API 只读接口
- Go Mock API 写接口：
  - 创建实例
  - 发布配置
  - 触发备份

## 当前仍未实现

### 1. 生产级后端能力

- 数据持久化
  - 目前所有数据都在内存中，服务重启后丢失
  - 下一步应接 PostgreSQL，并把 Mock 仓库替换成 Repository 层
- 异步任务队列
  - 当前创建实例、发布配置、触发备份都即时完成
  - 下一步应引入 `operation_job` 状态流和后台 worker
- 真实运行时编排
  - 当前没有接 Kubernetes / Docker Runtime Adapter
  - 下一步应把实例创建、配置发布、备份触发改为调用运行时与存储层

### 2. 认证与权限

- 真实登录
  - 当前登录页只是入口分流
  - 下一步应接 IAM、JWT、SSO 或企业身份系统
- 角色权限控制
  - 当前按钮和页面没有真正做权限收敛
  - 下一步应基于角色矩阵控制菜单、按钮与接口权限

### 3. Portal 缺口

- 实例详情 tab 化
  - 当前是单页纵向堆叠，信息够用但不够精细
- 配置编辑器
  - 当前只支持简单表单字段
  - 下一步应支持版本对比、回滚、校验结果展示
- 备份恢复
  - 当前只有触发备份，没有恢复流程
- 告警页与通知页
  - 当前只在概览中做摘要展示

### 4. Admin 缺口

- 集群与节点视图
  - 当前只保留了轻量聚合信息
- 远程终端与诊断
  - 当前完全未做
- 审批中心
  - 当前只有审计读取，没有审批流

### 5. 商业化与平台化能力

- 订单、订阅、支付、退款、发票
- 产品与套餐管理
- 到期治理与续费提醒
- 市场回调与支付回调

### 6. 测试与工程化

- 前端单元测试与组件测试
- Go handler 测试
- E2E 流程测试
- CI 构建与部署脚本

## 建议下一步开发顺序

1. 接数据库与 Repository 层，保留当前 API 形状不变
2. 把 `createInstance`、`updateConfig`、`backup` 改为真实异步任务
3. 为 Admin 增加实例详情页与集群/节点视图
4. 为 Portal 增加配置回滚与备份恢复
5. 接真实登录与权限体系
6. 最后再接订单、支付与商业化流程
